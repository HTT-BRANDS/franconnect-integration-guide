🔄 FranConnect Integration - Cloud Flow Troubleshooting

Quick diagnostic guide for when Power Automate flows aren't working

⚡ 5 Quick Checks (Do These First!)

  1. Are both flows turned ON? (green check vs gray off)
  2. Is Office 365 connection showing "Connected"?
  3. Are HubSpot emails in Inbox (not Junk)?
  4. Are HubSpot workflows showing "On" with enrollment history?
  5. Do flows use correct trigger "When a new email arrives in a shared mailbox (V2)"?

🔴 Tier 1: Most Likely (80% of cases)

🚨 Check These First
1 Flows Not Turned On
  • Check: make.powerautomate.com → Solutions → Cloud flows tab
  • Look for "Off" (gray) vs "On" (green)
  • Fix: Click ... menu → Turn on
2 Connection Not Authenticated
  • Check: Data → Connections
  • Look for "Can't sign in"
  • Fix: Click "Fix connection" and re-authenticate
  • If still won't turn on: Share connection with "Can use" permission
3 Emails Landing in Junk
  • Check: Open shared mailbox Junk Email folder
  • Critical: Power Automate ONLY monitors Inbox, not Junk!
  • Fix: Go to security.microsoft.com/tenantAllowBlockList
  • Add hubspotemail.net as allowed sender
  • Alternative: Create mail flow rule with SCL=-1

🟡 Tier 2: Likely If Tier 1 Fails

⚠️ Secondary Checks
4 Wrong Email Trigger
  • Must use: "When a new email arrives in a shared mailbox (V2)"
  • NOT: "When a new email arrives (V3)"
5 Environment Variables Wrong
  • Check: Solutions → Environment variables
  • Verify 4 email addresses (no typos)
6 Permissions Not Propagated
  • Wait 2 hours after granting Full Access
  • Then delete/recreate connection
7 HubSpot Workflows Not Running
  • Check: Automation → Workflows showing "On"
  • Check History tab for enrollments

🟢 Tier 3: Less Likely

🔍 Advanced / Edge Cases
  • Conditional Access / MFA issues
  • OAuth consent blocked
  • Solution import failed
  • ZAP (Zero-hour Auto Purge)

📋 Quick Decision Table

Symptom Most Likely Issue
Flows show "Off" Issue #1
"Can't sign in" Issue #2
Emails in Junk Issue #3
No emails at all Issue #7
Flow runs but no action Issue #4 or #5
