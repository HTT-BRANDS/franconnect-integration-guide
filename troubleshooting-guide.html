What's Actually Happening

HubSpot notification emails aren't just plain text — they contain HTML with logo images, tracking pixels, styled links, and a footer full of branding URLs. When Power Automate's Strip HTML action runs, it strips the tags but keeps the image descriptions (alt text) and converts links to href URLs in brackets. The result is a messy string where the actual field values are buried between image descriptions and CDN URLs.

Why the Current Expression Is Wrong

The current Compose - First Name action is using first(split(...)) which takes everything before the label delimiter — that's the HubSpot header content, not the value. Even if you fix the index, a single split returns everything from the label to the end of the entire email, including all other fields and the HubSpot footer.

cancel Current (Wrong) Expression
trim(first(split(body('Strip_HTML'), 'First Name: ')))
Problem: first() returns index 0 — everything BEFORE "First Name:", not after. Gets header junk.
check_circle Fixed (Two-Stage Split)
trim(first(split(last(split(body('Strip_HTML'), 'First Name: ')), 'Last Name: ')))
How it works: Step 1 isolates everything after "First Name:". Step 2 cuts at the next label. What's left is just the value.

❌ Split returns empty / blank value

The delimiter string in your expression doesn't match the actual text. Open the Strip HTML output, copy the exact characters before and after the value you want, and update your expression to match.

❌ Value includes a newline or extra space

The trim() wrapper handles leading and trailing spaces, but not embedded newlines. If your value still has extra characters, try replacing trim(...) with replace(trim(...), decodeUriComponent('%0A'), '')

❌ Last field (State) still includes footer content

State is the last field before the HubSpot footer. Splitting on 'Record Type: ' as the end delimiter works only if "Record Type: Lead" appears after State in the stripped email. Check this in the actual Strip HTML output. If the order is different, adjust the delimiter accordingly.

❌ HubSpot updated their email template and it broke again

This is a known fragility of email-parsing approaches. If HubSpot changes their notification email layout, the labels may shift or new content may be injected. See Option F below — sending a plain-text structured email eliminates this fragility.

❌ Expression editor in Power Automate clears when I click elsewhere

This is a known Power Automate UI behavior. Type or paste the expression in the Expression tab, then immediately click OK — do not click anywhere else first. If the expression disappears, paste it again.

Send a Plain-Text Structured Email from HubSpot

Reconfigure the HubSpot workflow to send a clean plain-text email with predictable formatting. Eliminates image noise and makes the expressions reliable — no more guessing about labels.

The root problem isn't the expressions — it's that HubSpot's default notification emails are full of HTML branding, images, and tracking pixels that pollute the stripped text. If you control what HubSpot sends, you can send a clean, predictable plain-text email that always looks exactly the same.

How to Set This Up in HubSpot

  1. 1
    Go to HubSpot → Automation → Workflows — open the Bishops workflow that currently sends the notification email
  2. 2
    Find the "Send internal email notification" action — click on it to open the action editor
  3. 3
    Replace the email body with the plain-text template below — use the HubSpot personalization token picker to insert each contact property
  4. 4
    Set the email format to plain text — look for a "Format" or "Send as" option in the action and select "Plain text" if available. If not, use a minimal template with no images.
  5. 5
    Save and test — manually enroll a test contact, then check the flow run history to confirm the Strip HTML output now shows clean label: value lines

Skip Email Parsing — Use FranConnect's REST API or Marketplace Connector

Push lead data directly from HubSpot to FranConnect via API — no email, no parsing, no fragility. The most reliable architecture long-term.

The entire email-parsing chain (HubSpot → email → shared mailbox → Strip HTML → split expressions → FranConnect email) exists because there was no direct integration available at the time. FranConnect has a REST API and a HubSpot Marketplace connector that, if available on your plan, would eliminate the intermediary steps entirely.
